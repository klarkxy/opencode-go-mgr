# Security Policy / 安全政策

## Reporting a Vulnerability

Report security issues privately to `278370456@qq.com`. Include the affected
version and platform, reproduction steps or a proof of concept, the observed
impact, and sanitized logs or screenshots when useful.

Do not open a public issue before disclosure is coordinated. Never send
OpenCode-Go account keys, Gateway Keys, administrator credentials, access
tokens, database files, or other secrets. If a secret was exposed, revoke or
rotate it first.

## Supported Versions

Security fixes target the latest published release and the current default
branch. Reports for older versions may require upgrading before investigation.
The project is maintained on a best-effort basis and does not guarantee a
specific response or resolution time.

## 报告安全漏洞

请通过 `278370456@qq.com` 私下报告安全问题。至少提供受影响的版本与平台、
复现步骤或概念验证、实际影响；必要时可附上已脱敏的日志或截图。

在协调披露前，请勿创建公开 Issue。不要发送 OpenCode-Go 账号 Key、Gateway
Key、管理员凭据、访问令牌、数据库文件或其他秘密。如果秘密已经泄露，请先
撤销或轮换。

## 支持范围

安全修复以最新公开版本和当前默认分支为目标。旧版本的问题可能需要先升级
再调查。本项目按维护者可用时间尽力处理，不承诺具体响应或修复时限。
